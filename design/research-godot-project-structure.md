# Research: Godot 4.x .NET Project Structure

## 1. Mandatory Files

**`project.godot`** — Project definition file. Must be at the root of the Godot project folder. Godot editor requires this to recognize a directory as a project. Key C# setting: `dotnet/project/assembly_name`.

**`*.csproj`** — Uses `Godot.NET.Sdk` instead of `Microsoft.NET.Sdk`. The SDK version must exactly match the installed Godot editor version (e.g., `Godot.NET.Sdk/4.6.1`). Requires `<EnableDynamicLoading>true</EnableDynamicLoading>` for hot-reload.

**`*.sln`** — Godot auto-generates one alongside `project.godot`. It gets regenerated whenever scripts change, so it should be treated as generated/gitignored.

**No mandatory scene file** to build, but you need at least one `.tscn` to run.

## 2. Generated Files / .gitignore

**`.godot/` directory** (replaces Godot 3's `.import/` and `.mono/`): Contains `imported/`, `editor/`, `shader_cache/`, `uid_cache.bin`, `project.binary`. Always gitignore. Fully regenerated by editor.

**`.mono/`** — Godot 3 only. Not used in Godot 4.x. Godot 4 uses standard `bin/` and `obj/` (already in our `.gitignore`).

Additions needed to `.gitignore`:
```gitignore
# Godot 4.x
.godot/
*.translation

# Godot-generated solution
src/Pockets.Godot/*.sln
```

Files to **commit**: `project.godot`, `*.csproj`, `*.tscn`/`*.tres`, `*.cs`, `*.gdshader`, `export_presets.cfg`.

## 3. Build Integration — Two Solution Files (Recommended)

The Godot `.csproj` uses `Godot.NET.Sdk` which downloads from NuGet but references GodotSharp assemblies bundled with the local Godot editor installation. **`dotnet build` will fail without Godot installed.**

Therefore: **do NOT add the Godot .csproj to `Pockets.sln`**. Keep two solutions:

| Solution | Contains | Requires Godot? |
|----------|----------|-----------------|
| `Pockets.sln` (repo root) | Core, App, Tests | No |
| `Pockets.Godot.sln` (Godot-generated, gitignored) | Godot + Core ref | Yes |

`ProjectReference` from Godot to Core works normally since Godot.NET.Sdk uses standard MSBuild:
```xml
<ProjectReference Include="..\Pockets.Core\Pockets.Core.csproj" />
```

Target framework: Godot 4.3+ uses `net8.0`, matching our Core project.

## 4. Godot CLI Commands

```bash
godot --headless --build-solutions --quit --path src/Pockets.Godot  # Build C#
godot --path src/Pockets.Godot                                       # Run project
godot --headless --export-release "Linux/X11" /path/to/output        # Export
dotnet build src/Pockets.Godot/Pockets.Godot.csproj                  # Direct dotnet (needs Godot SDK)
```

## 5. Project Location — `src/Pockets.Godot/` (Recommended)

`project.godot` defines the resource root (`res://`). The `.csproj` must be in the same directory.

Recommended layout (matches stage-4b design doc):
```
pockets-godot/
├── src/
│   ├── Pockets.Core/
│   ├── Pockets.App/
│   └── Pockets.Godot/
│       ├── project.godot
│       ├── Pockets.Godot.csproj
│       ├── Scenes/Main.tscn
│       ├── Scripts/*.cs
│       └── .godot/              (gitignored)
├── tests/
├── data/
└── Pockets.sln                  (no Godot project)
```

**Data directory note**: `data/` is outside `res://`, so Godot's resource system cannot access it. Core already loads data via filesystem paths, so this works if the working directory or a configurable base path is set correctly.

## 6. C# Namespace Conventions

Godot imposes **no namespace requirements**. Use whatever you want (`Pockets.Godot`, etc.). One hard requirement: all node scripts must use `partial class` because the Godot source generator creates companion partial classes for signal/export wiring.

## 7. Key Configuration Files

**Minimal `Pockets.Godot.csproj`**:
```xml
<Project Sdk="Godot.NET.Sdk/4.6.1">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
    <EnableDynamicLoading>true</EnableDynamicLoading>
    <RootNamespace>Pockets.Godot</RootNamespace>
  </PropertyGroup>
  <ItemGroup>
    <ProjectReference Include="..\Pockets.Core\Pockets.Core.csproj" />
  </ItemGroup>
</Project>
```

**Minimal `project.godot`**:
```ini
[application]
config/name="Pockets"
config/features=PackedStringArray("4.4", "C#", "Forward Plus")
run/main_scene="res://Scenes/Main.tscn"

[dotnet]
project/assembly_name="Pockets.Godot"
```

## 8. Open Questions

1. **SDK version**: `Godot.NET.Sdk/4.6.1` — verify this exact version exists on NuGet. The SDK version must match the editor version precisely.
2. **Data path resolution**: Need a small abstraction so Core can find `data/` whether launched from TUI (repo root CWD) or Godot (project CWD is `src/Pockets.Godot/`).
3. **Hot reload across projects**: Changes to `Pockets.Core` may not trigger Godot's hot reload automatically — needs testing.
4. **CI/CD**: Keep Godot project out of `Pockets.sln` so `dotnet test` works without Godot in CI.
